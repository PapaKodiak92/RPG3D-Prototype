# Teaching Plan

This project should be fun to build and useful for learning programming.

The teaching style is simple:

1. Build something visible.
2. Explain the coding idea behind it.
3. Let the learner change a value or add a small feature.
4. Test it in the game.
5. Save the progress.

---

## Lesson format

Each lesson should follow this structure:

```text
Lesson title
What we are building
What coding idea it teaches
Files we will touch
Step-by-step build
What to test
Kid challenge
Parent notes
```

---

## Beginner-friendly rules

- Use simple names.
- Avoid huge files.
- Avoid clever tricks.
- Prefer obvious code over fancy code.
- Add comments when a concept is new.
- Make each lesson produce something visible in-game.

---

## Suggested first lessons

### Lesson 01: The First Scene

Coding ideas:

- Project structure
- Scenes
- Nodes
- 3D objects

Build:

- A ground plane
- A player placeholder
- A camera

### Lesson 02: Click-to-Move

Coding ideas:

- Variables
- Functions
- Mouse input
- Raycasting
- Vectors

Build:

- Click ground to choose a destination
- Player walks toward the clicked spot

### Lesson 03: Camera Follow

Coding ideas:

- Following another object
- Offsets
- Smooth movement

Build:

- Camera follows the player

### Lesson 04: Clickable Tree

Coding ideas:

- Objects
- Functions
- Conditions
- Interaction

Build:

- Click tree
- Gain one log

### Lesson 05: Basic Inventory

Coding ideas:

- Dictionaries/maps
- Item names
- Counts

Build:

- Track log count
- Show inventory text

### Lesson 06: Woodcutting XP

Coding ideas:

- Numbers
- Addition
- Level checks

Build:

- Chopping gives XP
- XP can increase a level

---

## Parent teaching notes

Good questions to ask during coding:

- What do you think this variable controls?
- What happens if we change this number?
- What should happen when the player clicks this object?
- What should the game remember?
- How can we test that this works?

The goal is not to rush. The goal is for each piece to make sense.
